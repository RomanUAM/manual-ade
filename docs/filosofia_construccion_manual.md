# Filosofia de construccion del manual

El objetivo del proyecto no es recopilar documentos, sino construir una experiencia de aprendizaje.

Todo el material existente constituye materia prima. El producto final no debe conservar la estructura de los documentos originales; debe reorganizar la informacion para que el estudiante aprenda de manera progresiva, intuitiva y significativa.

El lector nunca debe percibir que el contenido proviene de distintas practicas, presentaciones, examenes o apuntes. El manual debe leerse como una unica obra escrita especificamente para la asignatura.

## Organizacion del conocimiento

No organizar el contenido por documentos, archivos ni practicas existentes.

Organizarlo como un disenador instruccional.

Cada capitulo debe responder una pregunta de aprendizaje y preparar el siguiente.

## Jerarquia del aprendizaje

Cada capitulo seguira esta estructura:

1. Motivacion.
2. Historia.
3. Intuicion.
4. Fundamento teorico.
5. Ejemplo guiado.
6. Practicas.
7. Errores frecuentes.
8. Casos reales.
9. Actividades.
10. Conexion con el siguiente capitulo.

## Diseno editorial

El documento no debe parecer coleccion de diapositivas, apuntes ni tesis. Debe parecer un libro universitario moderno.

Cada pagina debe ensenar una idea. Cada figura debe explicar algo. Cada tabla debe sintetizar informacion. Cada fotografia debe aportar evidencia. Cada ejemplo debe responder una pregunta.

Si una pagina parece llena de texto, debe dividirse. Si parece vacia, debe enriquecerse. Si una figura no aporta aprendizaje, debe eliminarse.
